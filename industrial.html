Fleet Vision

From One Robot to Many

The system is architected to scale from a single RL rover to a coordinated fleet managed by a ROS 2 distributed discovery server. Each robot runs the same policy — swarm coordination emerges from shared world knowledge and local sensing.

Deployment Roadmap

Phase 1 — Edge

On-Device Acceleration

Move policy inference from CPU to a Coral USB Accelerator or TensorRT runtime. Target: sub-20 ms latency, freeing CPU for full sensor fusion and camera processing.

Phase 2 — Fleet

Multi-Agent Coordination

Scale to a coordinated fleet via ROS 2's distributed discovery server. Shared policy weights, decentralised collision avoidance, and dynamic task allocation across all nodes.

Phase 3 — Sim-to-Real

Domain Randomisation

Train with injected sensor noise, wheel slip, and variable lighting to close the sim-to-real gap — enabling zero-shot deployment into new warehouse environments without fine-tuning.
