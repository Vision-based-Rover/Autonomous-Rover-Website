Bill of Materials  —  Hardware Stack
Design Philosophy: Built on affordable, off-the-shelf components — delivering industrial-grade navigation without the enterprise price tag. Total BOM cost is a fraction of conventional AGV systems.

// Hardware Topology

Processing Node

Raspberry Pi 4 (4GB) — Primary host running ROS 2, Tiny-YOLO inference, and RL policy execution.

  • L298N Motor Driver: PWM-based differential wheel control
  • Differential Drive Chassis: High-maneuverability mobile base
  • 5V/3A Buck Converter: Regulated onboard power rail
ROS 2 Humble 10–20 Hz Inference

Perception Suite

Multi-modal sensor fusion for robust environment awareness and real-time obstacle detection.

  • VL53L0X ToF Sensors ×4 — Precision ranging up to 2m
  • MPU-6050 IMU — 6-axis orientation for self-correction
  • Wheel Encoders — Odometry feedback for spatial awareness
  • Sharp IR Sensors ×2 — Short-range proximity & wall following
Sensor Fusion EKF Odometry

Vision Hardware

Pi Camera Module 2/3 — Captures raw video feeds for vision-based navigation and object detection pipelines.

  • Tiny-YOLO inference for shelf/obstacle classification
  • ESP32-CAM optional for secondary viewing angles
  • OpenCV pipeline for lane and marker detection
Optional: ESP32-CAM

// System Redundancy

Proximity Alternatives

Ultrasonic HC-SR04 can substitute IR sensors for basic obstacle avoidance — enabling lower-cost prototype builds while maintaining functional parity.

Fallback Mode

Edge Inference

Future path: migrate RL policy inference to Coral USB Accelerator or TensorRT to achieve sub-20ms latency and free the Raspberry Pi CPU.
