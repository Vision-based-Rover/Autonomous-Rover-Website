System Design Philosophy

The system is designed for modularity, scalability, and real-time performance. Each component operates independently while maintaining a unified control pipeline.

// MVC_DESIGN_PATTERN

The codebase is refactored into a professional Model-View-Controller architecture to ensure modularity and testability.

Models (Business Logic)

  • agent.py: PPO wrapper for training/inference.
  • environment.py: Gymnasium manager for complex layouts.
  • training.py: Hyperparameter and path management.

Controllers (Orchestration)

  • TrainingController: Coordinates the curriculum learning workflow.
  • EvalController: Manages model validation and statistics.
  • ManualController: Facilitates WASD keyboard overrides.

Views (Presentation)

  • WarehouseRenderer: Abstract interface for Matplotlib/Pygame.
  • ConsoleDisplayer: Formatted terminal logging and status updates.

// DATA_FLOW

  • Sensors → Environment State Representation
  • State → RL Policy (PPO)
  • Policy Output → Velocity Commands
  • Safety Guard → Validated Actions
  • Motor Controller → Physical Execution

// ARCHITECTURE_BENEFITS

Why MVC Architecture?

Separating training logic, environment simulation, and rendering ensures maintainability, faster experimentation, and scalability to production systems.

// EMBEDDED_SOFTWARE_STACK

ROS 2 Hybrid Stack:

1. Sensor Bridge: Fuses ToF, IR, and IMU data.
2. State Estimator: EKF-based wheel odometry + IMU yaw.
3. Policy Node: 10-20Hz RL inference (PPO v6.2).
4. Safety Guard: Deterministic override for collision prevention.
