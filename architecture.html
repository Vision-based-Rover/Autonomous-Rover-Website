Software Design  —  System Architecture
Design Philosophy: Modularity, scalability, and real-time performance. Each component operates independently while maintaining a unified control pipeline. The MVC pattern ensures clean separation of concerns across training, evaluation, and deployment contexts.

// MVC Design Pattern

Models — Business Logic

  • agent.py — PPO wrapper for training & inference
  • environment.py — Gymnasium manager for complex layouts
  • training.py — Hyperparameter & path management
  • safety.py — Deterministic constraint enforcement
Gymnasium Stable-Baselines3

Controllers — Orchestration

  • TrainingController — Curriculum learning workflow
  • EvalController — Model validation & statistics
  • ManualController — WASD keyboard override mode
  • ROSBridgeController — Hardware command relay

// Architecture Benefits

Why MVC?

Separating training logic, environment simulation, and rendering ensures maintainability, faster experimentation cycles, and a clear path to production deployment.

Testability

Each layer can be unit-tested independently. Mock environments enable policy validation without physical hardware, accelerating iteration speed.

Extensibility

Adding a new sensor, renderer, or controller requires zero changes to the core training loop — just implement the abstract interface and plug it in.
