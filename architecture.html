Software Design — Control Architecture
The software is structured around a Model-View-Controller pattern and a layered ROS 2 node graph. The MVC layer and simulation stack are fully implemented. The ROS 2 embedded pipeline is designed and ready for hardware integration once the physical build begins.

Planned Real-Time Control Pipeline

Hardware-to-Motor Signal Flow

Design Status: This pipeline is fully designed and validated in simulation. ROS 2 node wiring will be implemented during hardware integration.
ToF / IR / IMU
Raw sensor readings will arrive at up to 50 Hz via I²C and GPIO. The Sensor Bridge Node normalises and time-stamps each reading before publishing to the ROS 2 graph.
EKF State Estimator
Wheel encoder odometry and IMU yaw will be fused through an Extended Kalman Filter at 100 Hz, producing a smooth pose estimate that corrects for wheel slip and drift.
PPO Policy Node
The PPO v6.2 policy — trained and validated in simulation — receives the fused state vector and outputs a continuous velocity command at 10–20 Hz. Inference runs entirely on the Raspberry Pi with no network dependency.
Safety Guard Node
A deterministic override layer clips any velocity command that would result in a collision given the current braking distance. This layer operates independently of the RL policy and cannot be disabled.
L298N Motor Driver
Safe, validated PWM signals are written to the motor driver. Left and right wheel speeds are set independently, enabling smooth differential steering through tight aisles.
