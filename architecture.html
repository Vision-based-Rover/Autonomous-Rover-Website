Software Design — Control Architecture
The software is structured around a Model-View-Controller pattern and a layered ROS 2 node graph. Navigation input comes from a 360° LiDAR; box and item identification runs on a dedicated camera recognition pipeline. The MVC layer and simulation stack are fully implemented. The ROS 2 embedded pipeline is designed and ready for hardware integration once the physical build begins.

Planned Real-Time Control Pipeline

Hardware-to-Motor Signal Flow

Design Status: This pipeline is fully designed and validated in simulation. ROS 2 node wiring will be implemented during hardware integration.
360° LiDAR / IMU
Raw LiDAR scans (~8,000 points per revolution at 10 Hz) and IMU readings arrive via USB and I²C. The Sensor Bridge Node normalises, time-stamps, and publishes each scan to the ROS 2 graph as /awr/scan and /awr/imu.
EKF State Estimator
Wheel encoder odometry, IMU yaw, and LiDAR-derived displacement are fused through an Extended Kalman Filter at 100 Hz, producing a smooth pose estimate that corrects for wheel slip and rotational drift.
PPO Policy Node
The PPO v6.2 policy — trained and validated in simulation — receives the fused state vector (pose + LiDAR obstacle map) and outputs a continuous velocity command at 10–20 Hz. Inference runs entirely on the Raspberry Pi with no network dependency.
Safety Guard Node
A deterministic override layer clips any velocity command that would result in a collision given the current LiDAR braking distance. This layer operates independently of the RL policy and cannot be disabled.
L298N Motor Driver
Safe, validated PWM signals are written to the motor driver. Left and right wheel speeds are set independently, enabling smooth differential steering through tight aisles.
Parallel Camera Pipeline — Box & Item Recognition: The Pi Camera runs a dedicated Tiny-YOLO recognition node in parallel with the navigation pipeline. It publishes detected box labels, SKUs, and item classes to /awr/items_detected — completely decoupled from the LiDAR navigation stack so neither pipeline blocks the other.
